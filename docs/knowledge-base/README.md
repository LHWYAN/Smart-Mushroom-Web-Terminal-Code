# 知识库文档说明

本目录下的 Markdown 文件用于导入 **Dify 知识库**，供「种植问答」功能检索回答。

## ⚠️ 导入 Dify 若提示 `file not find`

在 **Windows + Docker 版 Dify** 中较常见，通常由以下原因导致：

| 原因 | 说明 | 解决办法 |
|------|------|----------|
| **中文文件名** | 上传后 Worker 嵌入时找不到原文件 | ✅ 使用本目录下 **英文文件名** 的版本 |
| **填写本地路径** | Dify 跑在 Docker 里，读不到 `D:\...` 路径 | ✅ 用浏览器 **上传文件**，不要填路径 |
| **上传未完成就点嵌入** | 文件尚未写入存储卷 | ✅ 等上传进度 100% 后再点「开始索引」 |
| **旧文档残留** | 之前失败的上传记录损坏 | ✅ 删除失败文档后重新上传 |

**推荐上传这两个文件（英文文件名）：**

- `01_mushroom_environment_guide.md`
- `02_mushroom_cultivation_faq.md`

**备用方案（100% 可用）：** Dify 知识库 → 添加文档 → **通过文本创建** → 复制粘贴 `.md` 全文 → 保存并索引。

## 导入 Dify 步骤

1. 登录 Dify → 进入知识库（「智慧蘑菇农场 AI 助手」关联的知识库）
2. **添加文档** → **上传文件**（不要选「本地目录同步」）
3. 选择 `01_mushroom_environment_guide.md` 和 `02_mushroom_cultivation_faq.md`
4. 分段：自动分段，500～800 字符
5. 点击 **保存并处理** / **开始索引**，等待状态变为「已完成」

若仍失败，重启 Dify 后再试：

```powershell
cd D:\LHWYAN\program\neu\ai\dify\docker
docker compose restart worker api
```
